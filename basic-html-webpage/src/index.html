CONOCIMIENTO DÉBIL. CUESTIONES SOBRE DISEÑO TECNOLÓGICO

Este trabajo busca poner de manifiesto algunos rasgos no-técnicos del diseño tecnológico, especialmente en la actividad que genera aquello que funciona por primera vez. Para ello se busca comparar la idea de ciencia aplicada con algunos rasgos que resultarían incompatibles con esta perspectiva, proponiendo el concepto de "conocimiento débil" para describir la instrumentalización del conocimiento por parte de los diseñadores. Se trata de un trabajo introductorio que no pretende presentar una conclusión cerrada, sino más bien de un disparador de la discusión y el debate que debe ser completado por epistemólogos y experiencias de diseño en ingeniería.

A deep learning approach for detecting malicious JavaScript code

El código JavaScript malicioso en páginas web de Internet es un problema de seguridad emergente debido a su universalidad y su impacto potencialmente grave. Debido a su ofuscación y complejidad, detectarlo tiene un coste considerable. En los últimos años, se han propuesto diversos enfoques de detección basados en aprendizaje automático; la mayoría de ellos utilizan modelos de discriminación superficial con características construidas con reglas artificiales. Sin embargo, con la llegada de la era del big data para la transmisión de información, estos métodos existentes ya no pueden satisfacer las necesidades reales. En este artículo, presentamos un nuevo marco de aprendizaje profundo para la detección de código JavaScript malicioso, del cual obtuvimos la mayor precisión de detección en comparación con el grupo de control. .

Servidores Web

Principales servidores web disponibles en el mercado para infraestructura tecnológica

  • Apache HTTP Server - Servidor web de código abierto más popular del mundo, desarrollado por la Apache Software Foundation
  • Nginx - Servidor web y proxy inverso de alto rendimiento, especialmente eficiente para sitios web con alto tráfico
  • Microsoft IIS (Internet Information Services) - Servidor web extensible de Microsoft para Windows Server
  • LiteSpeed Web Server - Servidor web comercial de alto rendimiento con compatibilidad nativa con Apache
  • Caddy - Servidor web moderno con HTTPS automático y configuración simplificada
  • Node.js (con Express) - Entorno de ejecución para JavaScript del lado del servidor, ideal para aplicaciones web dinámicas
  • Tomcat - Servidor de aplicaciones Java desarrollado por Apache, especializado en servlets y JSP
